    # Tablet-ready (swipe) to select albums and read albums
    # Each author can brand texTility (colors, logo and even domain name)
    # Each album can be either shared across all the eco-system or visible only

    # Each album can be either private, public or accessible when the visitor logs in
    # Each album can have a cover page
    # Albums can be created automatically from a local directory: each JPG files is then turned into a page.
    # Albums can also be created from a multi-page PDF files: each page is then turned into a JPG file which is inserted

    # Each album can be released for public viewing at a certain date
    # The concept of first paid page exists and can be enabled on-demand

    # Pages are created as JPG files
    # Each page has a description attached to it
    # Each page has a released date which is core to the concept of sequential publishing
    # Page can be re-organized with a drag/drop feature
    # Strong caching to speed up page presentation
    # Cropping feature to create very specific thembnails
    # Album easy PDF printing
    # Search by album or author

    # Automatic thumbnail creation for all pages
